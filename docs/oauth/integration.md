# OAuth Integration

The OAuth integration is designed to allow third-party services to link their accounts to OmniLink profiles using the OAuth 2.0 protocol. It is responsible for generating and verifying OAuth tokens.

OmniLink represents the primary authorization server for all OAuth flows, and it is responsible for generating and verifying OAuth tokens.

## Device flow

OmniLink supports the OAuth 2.0 Device Flow for account linking with third-party services. This flow is designed for games (like Minecraft) and other applications that do not have a web browser or cannot easily display a web page for user authentication.

1. The user initiates the device flow by interacting with the third-party service
2. The third-party service generates a unique device code and user code, and displays the user code to the user along with instructions to visit an OmniLink URL on a web browser.
3. The user visits the OmniLink URL on a web browser and enters the user code to authenticate and authorize the third-party service to access their OmniLink profile.
4. OmniLink generates an access token and refresh token for the third-party service, and returns them to the third-party service along with an expiration time for the access token.
5. The third-party service can use the access token to access any OmniLink API endpoints on behalf of the user, such as account linking and management.

The OmniLink API would require specific scopes to be requested by the third-party service in order to access certain endpoints. The scopes would be defined by OmniLink and would determine what actions the third-party service can perform on behalf of the user.

Access tokens generated by OmniLink have a limited lifespan and will expire after 3 minutes. The OmniLink API does not support refresh tokens for all scopes, to ensure that sensitive actions require the user to re-authenticate and authorize the third-party service.
